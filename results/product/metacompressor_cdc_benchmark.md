## Analysis

### Did CDC improve over fixed?

**Partially.**

| File | Result | Reason |
|------|--------|--------|
| empty.txt | N/A | No data; CDC container is slightly larger due to extra metadata fields stored in the header |
| small.txt | CDC **worse** (+0.1530) | Both modes produce output larger than the original; CDC adds extra metadata with no data to offset it |
| repeated.txt | CDC **better** (−0.0003) | Tiny win: the Gear hash finds a consistent natural split in the repeating 45-byte pattern, eliminating a boundary artifact of fixed 4 096-byte cuts |
| app.log | CDC **worse** (+0.0061) | Structured log lines vary in length; fixed 4 096-byte windows happen to align well with the repeating preamble. CDC fragments these windows into irregular sizes, reducing dedup hits |
| json_logs.log | CDC **better** (−0.0032) | CDC finds content-aware boundaries inside JSON records, aligning chunk cuts with structural repetitions that fixed chunking misses across boundary seams. Saves ≈ 1.3 KB |

### Did CDC beat ZSTD anywhere?

**No.** ZSTD's sliding-window LZ77 + entropy coding dominates on every test file.  
MetaCompressor's competitive edge requires data with high cross-file or cross-session chunk repetition — a scenario not captured by single-file compression benchmarks.

### Where did CDC lose?

- **Small files** (< min_chunk_size): CDC metadata overhead is fixed at ~30 bytes extra, which hurts when there is almost nothing to compress.
- **Highly structured fixed-stride logs** (app.log): Fixed-chunk alignment serendipitously matched the log structure; CDC broke those alignments.

### Was overhead worse on small files?

Yes. The CDC container header stores five extra fields (`chunking_mode`, `min_chunk_size`, `avg_chunk_size`, `max_chunk_size`, `cdc_mask`) that add ~30 bytes to the serialised payload.  
On `small.txt` (183 bytes) that overhead is visible as a 28-byte increase (189 → 217 bytes).

### Recommended next step

1. **Delta encoding / similarity matching**: For chunks that are *almost* identical (e.g., successive log lines that differ by only a timestamp), store a reference chunk plus a delta.  This is the single biggest lever available now that CDC boundary detection is in place.
2. **Stronger sequence compression**: The sequence of chunk-IDs is currently compressed by ZSTD as part of the msgpack payload; switching to a dedicated integer compressor (e.g., StreamVByte or Simple8b) could reduce container overhead on non-deduplicated files.
3. **Tune CDC window**: For structured text logs the current avg_size=4 096 may be too large.  A smaller avg (e.g., 1 024) could improve dedup hit rate on json_logs-like workloads.
